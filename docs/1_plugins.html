Plugins

Create a plugin including test-suite, readme with one-liner. hemera-cli
Please ensure that a plugin never extends another plugin that use the same topic name. This results in a plugin which can not be run alone because the publisher is not aware of that composition.

You can create a plugin in two different ways:

Create a plugin for publishing as package


exports.plugin = function myPlugin (hemera, opts, next) {
  const topic = "math"

  hemera.add({
    topic,
    cmd: "add"
  }, function (req, cb) {
    cb(null, req.a + req.b)
  })

  next()
}

exports.options = {
  name: require("./package.json").name
}
Use always the plugin helper to create new plugins hemera-plugin. It will provide a consistent interface and checks the bare-minimum version of Hemera.

Create a plugin inline


const myPlugin = function (hemera, opts, next) {
  const topic = "math"

  hemera.add({
    topic,
    cmd: "add"
  }, function (req, cb) {
    cb(null, {
      result: req.a + req.b
    })
  })

  next()
}

hemera.use({
    plugin: myPlugin,
    options: {
      name: "test"
    }
})

Plugin dependencies

Plugin dependencies are installed as usual with NPM.

Plugin registration

A plugin must be registered before the ready function is called. The ready function will initialize all plugins. Default plugin options are preserved if you don't overwrite them. 


hemera.use(plugin, [opts], [cb])

Default options.

Pass an object as second arguments to share default options.


  hemera.use({
    plugin: function(hemera, opts, next) { next() },
    options: { name: "foo" }
  }, { a: 1 })

Register multiple plugins at once


  hemera.use([{
    plugin: function(hemera, opts, next) { next() },
    options: { name: "foo" }
  }]).after((err) => { })

After

Calls a function after all the previously defined plugins are loaded, including all their dependencies.


  hemera.use(plugin).after(cb)
  // or
  hemera.use(plugin)
  hemera.after(cb)

Payload validators

Registered plugins can have different payload validators. This feature depends on the implementation of the validator. In Joi and Parambulator the option payloadValidator will be used to check which validator your plugin used. E.g when you don't set the payloadValidator in the options you can change it later in the corresponding plugin with:


hemera.setOption("payloadValidator", "hemera-joi")

Shared data

Sometimes it is useful to share data across plugins. This data can be accessed from all other plugins. You can also declare the dependencies of your function.


hemera.decorate("magicNumber", fn, ['dep1', 'dep2'])
hemera.magicNumber

Async / Await

You can also pass an async function to the use function.


  hemera.use({
    plugin: async function(hemera, opts) {
      return await Promise.resolve()
    },
    options: { name: "foo" }
  })
