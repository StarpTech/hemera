Basic

Synchronous errors

Hemera won't catch any thrown error expect you are using async/await. It's up to the developer to handle business specific issues. Ensure that your process is restarted e.g with a process-manager like PM2.


hemera.add({ topic: "math", cmd: "add" }, function (req, cb) {
  throw new Error()
})

Asynchronous errors

In asynchronous code, the application cannot catch exceptions as youâ€™ve lost your stack once you have entered a callback: 


hemera.add({ topic: "math", cmd: "add" }, function (req, cb) {
  try {
    process.nextTick(() => {
        throw new Error()
    })
  } catch(err) {

  }
})

"How can I deal with it?" Read this article about async error handling in express it's very useful.

Use callback function to propagate errors:


hemera.add({ topic: "math", cmd: "add" }, function (req, cb) {
  cb(new CustomError("Invalid operation"))
})

Hemera strictly follows the Error-first-callback convention.


hemera.act({ topic: "math", cmd: "add", a: 1, b: 1 }, function (err, resp) {
 err instanceOf CustomError // true
})

Use promises to propagate errors:


hemera.add({ topic: "math", cmd: "add" }, async function (req) {
  await operation()
  await operation2()
})

Create custom errors


const UnauthorizedError = Hemera.createError("Unauthorized")

hemera.add({
 topic: "math",
 cmd: "add"
}, function (req, cb) {
 const err = new UnauthorizedError("Unauthorized action")
 cb(err)
})

hemera.act({
 topic: "math",
 cmd: "add",
 a: 1,
 b: 2
}, function (err, resp) {
 console.log(err instanceof UnauthorizedError)
})
